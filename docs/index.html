ğŸ—ï¸ Architecture

SpendWise follows the MVVM (Model-View-ViewModel) architecture pattern, ensuring clean separation of concerns and maintainability.

Architecture Layers

  • View Layer: Activities and Fragments handle UI interactions
  • ViewModel Layer: Manages UI-related data and business logic
  • Model Layer: Data models and business entities
  • Repository Layer: Data access abstraction (Firebase integration)

Design Patterns Used

Singleton Pattern

Firebase database instance management

Factory Pattern

Chart creation and data aggregation

Strategy Pattern

Expense sorting strategies (by date, amount, category)

Command Pattern

Chatbot command handling (NEW in Sprint 4)
